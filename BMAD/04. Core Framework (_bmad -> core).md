BMAD-METHOD의 시스템 구조(System Architecture) 내에서 Core 프레임워크(`_bmad/core`)는 전체 프레임워크가 작동하기 위한 가장 밑바탕이 되는 핵심 인프라를 의미.

이는 단순한 폴더를 넘어, 설치부터 런타임 실행까지 AI 기반 개발 환경을 조율하는 시스템의 뼈대 역할.

## Core 프레임워크의 역할과 시스템 구조적 맥락


### 1 . 7계층 시스템 아키텍처의 오케스트레이터

- **진입 및 설치(Entry & Installation):** CLI를 통한 사용자 입력 처리, 모듈 탐색 및 의존성 해결, 설정 수집을 담당.

- **컴파일 및 IDE 통합(Compilation & IDE Layer):** <mark class="hltr-purple">YAML로 작성된 에이전트 정의를 마크다운(XML 포함)으로 컴파일</mark>하고, 20개 이상의 다양한 AI IDE(Cursor, Claude Code 등)에 맞는 플랫폼 종속적인 명령어를 템플릿을 통해 생성.

- **런타임 및 스토리지(Runtime & Storage):** 워크플로우 엔진을 통해 단계별 실행을 강제하며, 모든 산출물과 에이전트의 영구 메모리(Sidecar)를 저장소 계층에 관리.


### 2. 3계층 모듈 생태계의 최하단 기반 (The Foundation)

BMAD의 모듈 시스템은 <mark class="hltr-purple">Core(핵심)</mark>, <mark class="hltr-purple">Official Extensions(공식 확장)</mark>, <mark class="hltr-purple">Custom(사용자 정의)</mark>이라는 3계층 구조를 가짐.

- 이 중 `_bmad/core`에 설치되는 Core 모듈은 다른 어떤 모듈보다 <mark class="hltr-purple">항상 가장 먼저 설치</mark>됨.

### 3. 시스템 전체를 아우르는 범용 기능 및 태스크 제공

Core 프레임워크는 특정 도메인에 국한되지 않는, 시스템 전체가 사용하는 보편적인 기능과 에이전트를 내장.

- **지능형 라우팅 (bmad-help):** 현재 프로젝트의 상태를 스캔하고 <mark class="hltr-purple">다음에 실행할 적절한 워크플로우를 안내</mark>해 주는 메타 시스템.

- **공통 워크플로우 및 에이전트:** 여러 에이전트가 동시에 협업하는 파티 모드(Party Mode), 브레인스토밍, 문서 분할(Sharding) 태스크 등 <mark class="hltr-purple">다른 모듈들이 언제든 호출할 수 있는 범용 도구를 제공</mark>함.

- **계층적 환경 설정(Config Inheritance):** 사용자의 이름(`user_name`), 언어 설정(`communication_language`), 기본 출력 폴더(`output_folder`)와 같은 <mark class="hltr-purple">최상위 기본 구성 변수를 관리하여, 하위 모듈들이 이를 상속받아 일관되게 사용</mark>하도록 함.


### 4. AI의 접근성을 고려한 디렉토리 설계 (_bmad vs .bmad)

시스템 구조 설계에서 가장 중요한 결정 중 하나는 핵심 프레임워크의 저장 위치를 기존의 숨김 폴더(`.bmad`)에서 언더스코어(`_bmad`)로 변경한 것.

- 일반적으로 IDE의 인덱싱 시스템이나 대형 언어 모델(LLM)의 컨텍스트 윈도우는 점(`.`)으로 시작하는 숨김 폴더를 구성 파일로 간주하여 무시(필터링)하는 경향이 있음.

- 시스템 핵심 파일들을 `_bmad/core/`에 배치함으로써, AI 에이전트가 Core 프레임워크의 규칙, 컨텍스트, 에이전트 메모리 등에 자연스럽게 접근하고 이를 온전히 인지할 수 있도록 가시성을 확보.


## 결론

시스템 구조라는 큰 맥락에서 볼 때, Core 프레임워크(`_bmad/core`)는 다양한 AI IDE와 사용자 맞춤형 도메인 모듈(BMM, TEA 등) 사이를 연결해 주며, AI가 길을 잃지 않고 일관된 규칙과 컨텍스트 안에서 협업할 수 있도록 돕는 <mark class="hltr-purple">가장 강력하고 기본적인 운영체제(OS)와 같은 역할</mark>을 수행
