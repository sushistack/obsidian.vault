
BMAD-METHOD의 시스템에서 <mark class="hltr-orange">/bmad-help</mark> (지능형 내비게이션)은 단순히 도움말 텍스트를 출력하는 기능을 넘어, 전체 프레임워크를 조율하는 가장 핵심적인 '메타 도구(Meta-tool)'이자 '독립형 코어 유틸리티(Core Task)'로 기능


## `/bmad-help` 설계 및 작동원리


### 1. 상태가 없는 독립형 코어 유틸리티 (Stateless Standalone Task)

BMAD의 실행 단위는 크게 '에이전트(Agent)', '워크플로우(Workflow)', '태스크/도구(Task/Tool)'로 나뉨.

- 에이전트나 워크플로우가 여러 단계에 걸쳐 상태(State)를 유지하며 인간과 장기적인 협업을 수행하는 것과 달리, `/bmad-help`는 상태 추적 없이 단일 목적을 수행하는 독립형 태스크(Standalone Task)로 분류 됨.

- 특정 에이전트의 페르소나를 먼저 로드할 필요 없이 언제든 <mark class="hltr-orange">/bmad-help</mark> 슬래시 명령어를 통해 즉시 호출할 수 있는 범용 유틸리티. 이는 문서 분할 도구(<mark class="hltr-orange">/bmad-shard-doc</mark>)나 문서 색인 도구(<mark class="hltr-orange">/bmad-index-docs</mark>)와 같은 코어 유틸리티들과 동일한 작동 방식.


### 2. 프로젝트 상태 탐지기 (Project State Analyzer)

<mark class="hltr-orange">/bmad-help</mark>는 사용자가 현재 프로젝트의 어느 단계에 있는지를 시스템적으로 파악하는 진단 도구 역할.

- 명령어가 실행되면 시스템은 프로젝트 내의 `planning_artifacts/` 및 `implementation_artifacts/` 디렉토리를 스캔 함.

- <mark class="hltr-orange">product-brief.md</mark>, <mark class="hltr-orange">PRD.md</mark>, <mark class="hltr-orange">architecture.md</mark>, <mark class="hltr-orange">sprint-status.yaml</mark> 등의 아티팩트(결과물)가 존재하는지 확인하여, 현재 프로젝트가 Phase 0(사전 기획)부터 Phase 4(구현) 중 어느 단계에 도달했는지 정확히 추론해 냄.


### 3. 동적 카탈로그 검색기 (Dynamic Catalog Searcher)

유틸리티는 정적인 도움말을 제공하지 않고, 사용자가 설치한 모듈 환경에 맞춰 동적으로 작동하는 검색기

- <mark class="hltr-orange">/bmad-help</mark>는 Core, BMM은 물론 TEA(테스트 아키텍트), CIS(창의적 지능) 등 설치된 모든 모듈의 <mark class="hltr-orange">module-help.csv</mark> 매니페스트 파일을 읽어 들여 사용 가능한 워크플로우 카탈로그를 구축.

- 예를 들어, 사용자가 "이 기능을 어떻게 테스트할까?"라고 물었을 때, TEA 모듈이 설치되어 있다면 <mark class="hltr-orange">/bmad-tea-test-design</mark>과 같은 고도화된 테스트 전략 워크플로우를 추천하고, 없다면 BMM의 기본 <mark class="hltr-orange">/bmad-bmm-qa-automate</mark> 도구를 추천하는 식으로 설치된 환경에 맞춰 안내를 조율.


### 4. 자연어 기반의 지능형 라우터 (Natural Language Intelligent Router)

<mark class="hltr-orange">/bmad-help</mark> 는 인간의 자연어 질의를 시스템이 실행 가능한 슬래시 명령어로 번역해 주는 강력한 라우팅 도구.

- 사용자가 "/bmad-help 다음에 무엇을 해야 하지?" 또는 "/bmad-help 간단한 버그 수정이야"라고 자연어로 질문하면, 시스템은 키워드(예: 'next', 'quick', 'fix')와 앞서 파악한 프로젝트 상태를 결합하여 분석.

- 분석 결과에 따라 사용자가 IDE에서 즉시 클릭하거나 실행할 수 있는 정확한 슬래시 명령어(예: <mark class="hltr-orange">/bmad-bmm-create-architecture</mark>, <mark class="hltr-orange">/bmad-bmm-quick-spec</mark>)를 찾아 반환.


### 5. 규모 적응형 지능 (Scale-Adaptive Intelligence)의 구현체

이 도구는 단순한 길 안내를 넘어 프로젝트의 규모에 따라 프로세스를 적절히 축소하거나 확장하도록 권고하는 판단 유틸리티.

- 사용자의 질의나 프로젝트 구조를 분석하여 해당 작업이 규모가 작고 명확한 경우, 무거운 4단계 기획(PRD, 아키텍처 작성 등)을 건너뛰고 'Quick Flow(신속 개발)' 트랙(<mark class="hltr-orange">/bmad-bmm-quick-spec</mark>)으로 진입하도록 유도.

- 반면 대규모 프로젝트나 명시적인 기획이 필요한 경우에는 정규 'Full Planning' 트랙으로 안내.


## 결론

<mark class="hltr-orange">/bmad-help</mark>는 "단순히 기능을 나열하는 도움말 창이 아니라, 파일 시스템 스캔, 동적 모듈 인식, 자연어 분석을 결합하여 사용자가 <mark class="hltr-purple">거대한 프레임워크 안에서 길을 잃지 않고 다음 실행할 명령어를 정확히 찾을 수 있도록 돕는 가장 지능적인 중앙 통제 유틸리티</mark>"
