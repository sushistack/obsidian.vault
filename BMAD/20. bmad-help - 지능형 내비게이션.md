
BMAD-METHOD의 시스템에서 <mark class="hltr-orange">/bmad-help</mark> (지능형 내비게이션)은 단순히 도움말 텍스트를 출력하는 기능을 넘어, 전체 프레임워크를 조율하는 가장 핵심적인 '메타 도구(Meta-tool)'이자 '독립형 코어 유틸리티(Core Task)'로 기능


## `/bmad-help` 설계 및 작동원리


### 1. 상태가 없는 독립형 코어 유틸리티 (Stateless Standalone Task)

BMAD의 실행 단위는 크게 '에이전트(Agent)', '워크플로우(Workflow)', '태스크/도구(Task/Tool)'로 나뉨.

- 에이전트나 워크플로우가 여러 단계에 걸쳐 상태(State)를 유지하며 인간과 장기적인 협업을 수행하는 것과 달리, `/bmad-help`는 상태 추적 없이 단일 목적을 수행하는 독립형 태스크(Standalone Task)로 분류 됨.

- 특정 에이전트의 페르소나를 먼저 로드할 필요 없이 언제든 <mark class="hltr-orange">/bmad-help</mark> 슬래시 명령어를 통해 즉시 호출할 수 있는 범용 유틸리티. 이는 문서 분할 도구(<mark class="hltr-orange">/bmad-shard-doc</mark>)나 문서 색인 도구(<mark class="hltr-orange">/bmad-index-docs</mark>)와 같은 코어 유틸리티들과 동일한 작동 방식.


### 2. 프로젝트 상태 탐지기 (Project State Analyzer)

<mark class="hltr-orange">/bmad-help</mark>는 사용자가 현재 프로젝트의 어느 단계에 있는지를 시스템적으로 파악하는 진단 도구 역할.

- 명령어가 실행되면 시스템은 프로젝트 내의 `planning_artifacts/` 및 `implementation_artifacts/` 디렉토리를 스캔 함.
- `product-brief.md`, `PRD.md`, `architecture.md`, `sprint-status.yaml` 등의 아티팩트(결과물)가 존재하는지 확인하여, 현재 프로젝트가 Phase 0(사전 기획)부터 Phase 4(구현) 중 어느 단계에 도달했는지 정확히 추론해 냅니다
