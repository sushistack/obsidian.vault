
BMAD-METHOD 생태계에서 파티 모드(Party Mode)는 단순한 다중 채팅 기능을 넘어, 전체 시스템 어디서나 호출하여 복잡한 문제를 다각도로 분석하고 해결할 수 있도록 설계된 가장 강력한 범용 코어 워크플로우(Core Workflow)이자 유틸리티 도구.

도구 및 유틸리티(Tools and Utilities)'라는 더 큰 맥락에서 파티 모드가 어떻게 설계되고 작동하는지는 다음과 같은 4가지 핵심 특징으로 설명할 수 있음.


## 파티 모드의 설계 및 작동 원리


### 1. 모든 에이전트에 자동 주입되는 전역 유틸리티 (Auto-Injected Global Tool)

파티 모드는 특정 모듈이나 에이전트에 종속되지 않는 핵심(Core) 도구.

- 시스템의 에이전트 컴파일러(<mark class="hltr-orange">YamlXmlBuilder</mark>)는 에이전트를 마크다운으로 빌드할 때, 모든 에이전트의 메뉴 하단에 <mark class="hltr-orange">[PM] Party Mode: Start multi-agent collaboration</mark>이라는 명령을 자동으로 주입(Auto-Inject) 함.

- 즉, 사용자는 개발자가 에이전트 파일에 이 기능을 수동으로 코딩하지 않았더라도, 어떤 에이전트와 대화하든 원할 때 즉시 이 다중 에이전트 협업 유틸리티를 꺼내 쓸 수 있음


### 2. A/P/C 체크포인트를 통한 '서브프로세스(Subprocess)'

**도구** 워크플로우 실행 관점에서 파티 모드는 메인 프로세스를 돕는 유연한 '서브프로세스 도구'로 작동.

- BMAD의 주요 워크플로우들은 단계(Step)가 끝나는 결정적인 지점마다 사용자에게 A(심층 탐색), P(파티 모드), C(계속)로 구성된 표준화된 체크포인트 메뉴를 제공.

- 사용자가 'P'를 선택하면, 시스템은 즉시 <mark class="hltr-orange">{project-root}/_bmad/core/workflows/party-mode/workflow.md</mark>라는 코어 워크플로우를 서브프로세스로 호출함.

- 여러 에이전트로부터 작성 중인 명세서나 코드에 대한 비판적이고 협력적인 피드백을 받은 뒤, 시스템은 다시 원래 실행 중이던 워크플로우의 메뉴로 안전하게 복귀(Return)하여 작업을 이어갈 수 있게 해줌.


### 3. BMad Master 기반의 오케스트레이션 엔진 (Orchestration Engine)

파티 모드는 단순히 여러 AI가 무질서하게 떠드는 것이 아니라, 'BMad Master'라는 코어 에이전트가 조율하는 고도화된 오케스트레이션 도구.

- <mark class="hltr-orange">/bmad-party-mode</mark> 슬래시 명령어를 통해 단독 도구로 실행할 수도 있음.

- 이 도구가 실행되면 BMad Master는 PM, 아키텍트, 개발자, UX 디자이너 등 전체 AI 팀을 하나의 대화방으로 불러모음.

- 각 에이전트는 자신만의 고유한 페르소나와 원칙을 유지한 채 사용자의 메시지에 반응하며, 서로의 아이디어에 동의하거나 반박하고(예: 아키텍트는 분산 시스템의 안정성을 주장하고, PM은 출시 속도를 주장하는 식), 서로의 의견을 바탕으로 해결책을 발전시킴.


### 4. 복잡한 의사결정과 품질 검증을 위한 문제 해결 도구

유틸리티로서 파티 모드는 단일 에이전트의 시각으로는 해결하기 어려운 복잡한 엔지니어링 및 기획 문제를 다루는 데 최적화되어 있음.

- 트레이드오프(Trade-off)가 수반되는 중대한 아키텍처 결정, 창의적인 브레인스토밍, 장애 발생 후 원인을 다각도로 분석하는 포스트모템(Post-mortem), 그리고 스프린트 회고 등에서 다양한 관점의 전문가 피드백을 합성하는 강력한 검증 도구로 활용 됨.


## 결론

파티 모드(Party Mode)는, "단일 AI의 편향이나 시야각 제한을 극복하기 위해 언제든(Anytime) 호출할 수 있도록 시스템 전반에 내장되어 있으며, <mark class="hltr-purple">여러 도메인 전문가 에이전트들의 페르소나를 충돌시키고 융합하여 최고의 결과물을 도출해 내는 BMAD의 핵심 협업 엔진</mark>"
