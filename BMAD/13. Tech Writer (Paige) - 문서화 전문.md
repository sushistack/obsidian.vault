
BMAD-METHOD의 주요 에이전트(Agents) 생태계 내에서 테크 라이터(Paige)가 수행하는 <mark class="hltr-purple">'문서화 전문(Documentation)'</mark> 역할은 단순한 텍스트 작성을 넘어, <mark class="hltr-purple">지속적으로 학습하고 프로젝트의 지식을 큐레이션</mark>하는 매우 독특하고 필수적인 위치를 차지.


## Paige의 역할과 시스템적 특징

### 1. '지식 큐레이터(Knowledge Curator)'로서의 페르소나와 소통 방식

주요 에이전트들은 각자의 도메인 전문성에 맞는 명확한 페르소나를 갖음. 기획자(John)가 집요하게 "왜?"라고 묻고, 개발자(Amelia)가 파일 경로 위주로 극도로 간결하게 말하는 반면, Paige는 <mark class="hltr-purple">'인내심 있는 교육자(Patient educator)'의 정체성</mark>을 갖음. 

그녀는 CommonMark, DITA, OpenAPI에 정통한 전문가로서, 복잡한 개념을 친구에게 가르치듯 유추(Analogy)를 사용하여 쉽게 풀어냄. "다이어그램 하나가 수천 단어의 가치가 있다"는 원칙에 따라 명확성을 최우선으로 추구하며, <mark class="hltr-purple">인간 사용자와 다른 AI 에이전트들이 시스템을 쉽게 이해할 수 있도록 도움</mark>.


### 2. 에이전트 생태계 내 유일한 '상태 유지(Stateful)' 및 '학습' 에이전트 (Sidecar System)

주요 에이전트 생태계 아키텍처에서 Paige의 가장 큰 기술적 차별점은 <mark class="hltr-purple">영구 메모리(Sidecar) 시스템의 활용</mark>.


- PM(John), 아키텍트(Winston), 개발자(Amelia) 등 대부분의 BMM 에이전트들은 과거 세션의 불필요한 맥락에 영향을 받지 않고 명시된 산출물에만 집중하기 위해 메모리가 없는 무상태(`hasSidecar: false`)로 설계되었음.

- 반면, Paige는 BMM 에이전트 중 <mark class="hltr-purple">유일하게 사이드카를 가지는 에이전트(hasSidecar: true)</mark>입니다. 그녀는 <mark class="hltr-orange">_bmad/_memory/tech-writer-sidecar/documentation-standards.md</mark>라는 전용 폴더와 파일을 통해 세션이 종료되어도 사용자의 문서화 선호도와 프로젝트 고유의 스타일 규칙을 영구적으로 기억하고 학습 함.


### 3. 도메인 특화 문서화 및 시각화 워크플로우 지원

Paige는 단순히 지시받은 텍스트를 쓰는 것을 넘어, 기술 문서화에 특화된 고유의 메뉴(워크플로우) 시스템을 갖추고 있음.

- **WD (Write Document) & US (Update Standards):** 문서를 작성할 때 <mark class="hltr-purple">사이드카 메모리에서 팀의 표준</mark>을 자동으로 불러와 적용하며, 새로운 규칙이 발견되면 'US' 명령을 통해 <mark class="hltr-purple">스스로 표준 지식베이스를 업데이트</mark>함.

- **MG (Mermaid Generate):** 텍스트만으로 설명하기 어려운 복잡한 기술적 흐름이나 아키텍처를 <mark class="hltr-purple">Mermaid 다이어그램 코드로 변환해 주는 시각화 기능을 제공</mark>함.

- **VD (Validate Document) & EC (Explain Concept):** 기존 문서가 프로젝트 표준을 잘 따르고 있는지 <mark class="hltr-purple">검증</mark>하고, 복잡한 개념을 <mark class="hltr-purple">명확한 예제</mark>와 함께 설명해 줌.


### 4. 단계(Phase)에 국한되지 않는 '전천후(Anytime)' 협업자

다른 주요 에이전트들은 보통 특정 단계에 종속되어 있음.  예를 들어 Mary(Analyst)는 Phase 1, John(PM)은 Phase 2, Bob(Scrum Master)은 Phase 4에서 주로 활약 함. 하지만 Paige는 <mark class="hltr-purple">'Anytime(언제든)' 호출할 수 있는 전천후 에이전트</mark>로 분류 됨. 

프로젝트 초기의 브라운필드 분석(<mark class="hltr-orange">document-project</mark>), 구현 중의 코드 설명, 배포 후의 API 문서화 등 소프트웨어 개발 수명 주기(SDLC) 전체에 걸쳐 지식을 체계화하는 크로스 커팅(Cross-cutting) 역할을 수행.


## 결론

Tech Writer(Paige)는 "일회성 작업을 수행하는 다른 에이전트들과 달리, 프로젝트의 문서화 규칙을 영구적으로 학습(Sidecar)하고, 복잡한 기술적 결과물들을 <mark class="hltr-purple">인간과 다른 AI가 모두 소화할 수 있는 명확한 표준 지식으로 번역해 내는 핵심적인 지식 보존자</mark>"
