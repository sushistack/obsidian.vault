
MAD-METHOD의 시스템 아키텍처에서 **IDE 통합(IDE Integration)**은 `npx bmad-method install` 명령어로 시작되는 전체 **'설치 및 통합(Installation & Integration)'** 파이프라인의 최종 목적지이자 핵심 단계.

이 프레임워크는 단순히 코드를 내려받는 것에 그치지 않고, 내부의 범용 아티팩트(에이전트, 워크플로우, 태스크)를 사용자가 선택한 AI 도구(Claude Code, Cursor, Windsurf 등)가 즉시 이해하고 실행할 수 있는 고유한 슬래시 명령어(Slash Commands)와 설정 파일로 변환하여 주입.


## 4가지 핵심 아키텍처적 특징


### 1. 매니페스트 및 템플릿 기반의 동적 명령어 생성 (Manifest-Driven Command Generation)

설치 파이프라인에서 모듈 복사와 에이전트 컴파일(YAML → Markdown)이 끝나면, IDE 통합이 바로 시작되지 않습니다. 먼저 `ManifestGenerator`가 설치된 모든 에이전트와 워크플로우를 스캔하여 `agent-manifest.csv`, `workflow-manifest.csv` 등의 카탈로그를 구축 함. 

이후 통합의 핵심 오케스트레이터인 `IdeManager`가 이 매니페스트 데이터를 읽어 들이고, 각 IDE 플랫폼에 맞는 템플릿(예: `default-agent.md`)에 변수를 주입하여 명령어 파일을 동적으로 생성해 냄. 

즉, IDE 통합은 하드코딩된 과정이 아니라 **"설치된 모듈의 상태(Manifest)를 소비하여 사용자 접점(명령어 파일)을 렌더링해 내는 템플릿 파이프라인"**.


### 2. 설정 주도형 설치 아키텍처 (Config-Driven Setup Architecture)


BMAD가 가장 강력하게 권장하는 '선호 IDE(Preferred IDEs)'인 Claude Code, Cursor, Windsurf는 복잡한 커스텀 설치 코드(`Custom Handlers`) 없이 통합됨.

- 대신 이들은 `platform-codes.yaml` 파일에 정의된 설정값만을 이용해 `ConfigDrivenIdeSetup`이라는 단일 공통 클래스를 통해 설치 됨.

- 이를 위해 시스템은 윈도우와 유닉스 환경의 호환성 문제를 막기 위해, 모든 계층적 경로 구조를 대시(`-`)로 연결된 평면적인 파일명(예: `bmad-agent-bmm-pm.agent.md`)으로 변환하는 강력한 경로 치환(Path Transformation) 시스템을 적용 함


### 3. 주요 IDE별 맞춤형 환경 구성

동일한 템플릿 생성기를 거치더라도, 설치 시스템은 각 IDE의 특성과 컨텍스트 로딩 방식에 맞춰 파일의 위치와 형식을 정밀하게 조정 함.

- **Claude Code:** Anthropic의 공식 CLI 도구로, `.claude/commands/bmad/` 디렉토리에 대시 기반의 평면적인 명령어 파일들을 생성합니다. 또한 영구적 컨텍스트를 위한 `project.json` 파일 생성과, 외부 도구 및 데이터 소스 접근을 위한 MCP(Model Context Protocol) 서버 설정도 통합 단계에서 지원할 수 있음.

- **Cursor:** AI-First 코드 에디터인 Cursor를 위해서는 프로젝트 수준의 AI 지시사항과 에이전트 페르소나 정보를 담는 `.cursorrules` 파일을 생성합니다. 이와 함께 `.cursor/commands/` 폴더에 슬래시 명령어를 주입하여, 사용자가 채팅창에서 자연어로 지시하거나 `/` 명령어로 직접 워크플로우를 호출할 수 있도록 완벽히 통합됨.

- **Windsurf:** Cascade 기반의 흐름에 최적화된 IDE로, `.windsurfrules`라는 전용 환경 설정 파일을 제공하며, 명령어 파일들은 `.windsurf/workflows/` 디렉토리에 위치시켜 IDE가 BMAD의 구조를 원네이티브하게 이해할 수 있도록 도움.


### 4. 안전한 업데이트와 상태 관리 (Update-Safe Maintenance)

'설치 및 통합' 맥락에서 볼 때, 사용자가 모듈을 추가하거나 커스텀 에이전트를 만들면 IDE 명령어는 새롭게 갱신되어야 함.

- 업데이트 시 `IdeManager`는 기존의 낡은 파일이나 사용자 환경을 망가뜨리지 않기 위해, 대상 디렉토리에서 `bmad` 접두사로 시작하는 파일만 선택적으로 지우는 '정밀 정리(Surgical cleanup)'를 먼저 수행 함.

- 또한 IDE별 설정값은 `_bmad/_config/ides/{ide-name}.yaml` 파일에 영구적으로 보존되므로(IDE Configuration Persistence), 사용자가 전체 프레임워크를 업데이트하더라도 IDE 통합 환경 설정을 매번 다시 입력할 필요가 없음.
