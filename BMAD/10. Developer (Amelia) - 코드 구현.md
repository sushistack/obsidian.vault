
BMAD-METHOD의 주요 에이전트(Agents) 생태계에서 개발자(Amelia)가 수행하는 '코드 구현(Code Implementation)'은 기획(PM)과 설계(Architect) 단계를 거쳐 세분화된 <mark class="hltr-purple">요구사항(Story)을 실제 작동하는 코드와 테스트로 변환하는 Phase 4(구현)의 핵심 엔진</mark>.


## Amelia의 코드 구현 역할


### 1. 컨텍스트 체인(Context Chain)의 엄격한 실행자 (Strict Executor)

Amelia는 스스로 새로운 기획을 하거나 구조를 뜯어고치는 에이전트가 아님. 그녀는 앞선 에이전트(PM John, Architect Winston, Scrum Master Bob)들이 작성하고 승인한 <mark class="hltr-purple">스토리 파일의 세부 사항과 팀 표준을 엄격하게 준수하는 '시니어 소프트웨어 엔지니어' 페르소나</mark>를 갖음. 

그녀의 작동을 제어하는 핵심 행동 규칙(Critical Actions)은 매우 단호. 코드를 구현하기 전에 반드시 전체 스토리 파일을 먼저 읽어야 하며, 자신이 원하는 대로 순서를 바꾸거나 건너뛰지 않고 <mark class="hltr-purple">작성된 순서대로 작업을 실행</mark>해야 합니다. 이는 앞 단계의 기획 및 기술 설계 의도가 구현 과정에서 임의로 변질되는 것을 막아주는 필수적인 통제 장치임.


### 2. 100% 무결성을 요구하는 테스트 주도 개발 (Test-Driven Quality)

Amelia가 수행하는 <mark class="hltr-orange">dev-story</mark> 워크플로우는 단순히 기능 구현에 그치지 않고, 자동화된 테스트와 결합.

- 모든 기존 테스트와 새로운 테스트가 100% 통과해야만 스토리를 리뷰 준비 상태로 간주.

- 각 세부 작업을 완료할 때마다 전체 테스트 스위트를 실행해야 하며, 실패하는 테스트가 있는 상태로는 절대 다음으로 넘어갈 수 없음.

- 특히 에이전트에게 "<mark class="hltr-purple">테스트가 작성되었거나 통과했다고 절대 거짓말하지 말 것</mark>"이라는 규칙을 강제하여 AI 특유의 환각이나 작업 회피를 방지.


### 3. `project-context.md` 와 `architecture.md`를 통한 전체 시스템 규범 준수

Amelia는 시스템의 다른 전문가 에이전트들이 만들어둔 규칙이라는 울타리 안에서 코드를 구현.

- <mark class="hltr-orange">dev-story</mark> 워크플로우를 실행할 때, 프로젝트의 코딩 스타일, 에러 처리 패턴 등이 담긴 <mark class="hltr-orange">project-context.md</mark>를 로드하여 기존 코드베이스의 일관성을 해치지 않음.

- 구현이 끝나면 자신의 코드를 검증하는 <mark class="hltr-orange">code-review</mark> 워크플로우를 수행하는데, 이때 <mark class="hltr-cyan">아키텍트(Winston)</mark>가 작성한 <mark class="hltr-orange">architecture.md</mark> <mark class="hltr-purple">문서를 로드하여 자신의 구현이 시스템 아키텍처 결정 사항을 제대로 준수했는지 대조하고 검증</mark>.


### 4. 극도로 간결한 소통 방식과 무상태성 (Stateless & Succinct)

시스템 아키텍처 관점에서 <mark class="hltr-cyan">Amelia</mark>는 <mark class="hltr-purple">별도의 영구 메모리(Sidecar)를 가지지 않는 무상태(hasSidecar: false)</mark> 에이전트로 설계됨.  이는 과거의 불필요한 대화 맥락에 영향을 받지 않고, 오직 스크럼 마스터(Bob)가 주입한 명확한 스토리 컨텍스트에만 집중하기 위함. 

또한, 그녀의 소통 방식은 다른 에이전트들과 확연히 다름. <mark class="hltr-red">불필요한 설명(fluff)을 배제</mark>하고, 파일 경로와 수용 기준(AC) ID 등 인용 가능한 정확한 문장으로만 말하는 <mark class="hltr-purple">초정밀(Ultra-succinct) 소통</mark>을 지향.


## 결론

Developer(Amelia)의 코드 구현은 "앞선 에이전트들이 형성한 기획과 아키텍처라는 견고한 뼈대 위에서, <mark class="hltr-purple">한 치의 오차나 건너뜀 없이 테스트로 검증된 코드를 생산</mark>해 내는 가장 규율 잡힌 마지막 실행 단계"
