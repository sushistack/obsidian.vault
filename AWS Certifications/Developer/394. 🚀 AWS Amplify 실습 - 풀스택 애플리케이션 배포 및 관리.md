
이번 글에서는 AWS Amplify를 사용하여 GitHub 리포지토리의 소스 코드를 가져와 웹 애플리케이션을 배포하고, Amplify 콘솔을 통해 백엔드 서비스를 관리하는 과정을 실습해 보겠습니다. Amplify는 CI/CD부터 데이터베이스, 사용자 관리까지 모든 것을 통합하여 제공합니다.

---

## 1. 🏗️ Amplify를 통한 애플리케이션 배포

1. **시작 프로젝트 클론**:
    
    - Amplify 콘솔에서 **'새로운 앱 생성(Create new app)'**을 클릭합니다.
        
    - Amplify가 제공하는 시작 프로젝트 템플릿(예: React 앱)을 GitHub 계정에 복제(clone)합니다.
        
2. **Amplify와 GitHub 연동**:
    
    - Amplify 콘솔로 돌아와 소스 코드가 있는 공급자(Provider)로 **GitHub**를 선택하고, Amplify가 GitHub 리포지토리에 접근하도록 권한을 부여합니다.
        
    - 복제한 리포지토리를 선택하고, 기본 빌드 설정을 그대로 둔 채 배포를 진행합니다.
        
3. **CI/CD 자동화**:
    
    - Amplify는 자동으로 CI/CD 파이프라인을 구축하여, CloudFormation을 통해 필요한 백엔드 리소스(예: CDK 툴킷)를 프로비저닝하고 애플리케이션을 빌드하여 배포합니다.
        
    - 배포가 완료되면, Amplify가 제공하는 도메인 URL을 통해 웹 애플리케이션에 바로 접속할 수 있습니다.

---

## 2. 📊 데이터 및 사용자 관리

Amplify 콘솔은 프런트엔드 애플리케이션 외에도 백엔드 서비스를 관리하는 직관적인 UI를 제공합니다.

### 1. 데이터 관리(Data Manager)

- **`데이터(Data)`** 메뉴에서 **`콘텐츠(Content)`**를 클릭하면, DynamoDB에 저장된 데이터를 직접 확인할 수 있습니다.
    
- 실습 앱에서 추가한 'to-do' 목록은 DynamoDB 테이블에 자동으로 저장되며, 이 데이터 관리자에서 바로 데이터를 추가하거나 수정할 수 있습니다.
    
- 데이터 관리자는 DynamoDB 테이블에 대한 추상화 계층 역할을 하므로, DynamoDB 콘솔로 직접 이동하지 않고도 데이터를 관리할 수 있습니다.

### 2. 사용자 관리(User Management)

- **`인증(Authentication)`** 메뉴는 사용자를 관리하기 위한 인터페이스입니다.
    
- 백엔드로 **Amazon Cognito**와 통합되어, 사용자 계정을 생성하고, 인증 및 권한 설정을 관리할 수 있습니다.

---

## 3. ☁️ 백엔드 서비스 추상화

Amplify는 복잡한 AWS 인프라를 추상화하여 개발자가 비즈니스 로직에만 집중할 수 있게 해줍니다.

|Amplify 메뉴|백엔드 AWS 서비스|설명|
|---|---|---|
|**`저장소(Storage)`**|**Amazon S3**|파일 업로드 및 관리를 위한 백엔드 스토리지.|
|**`함수(Function)`**|**AWS Lambda**|서버리스 백엔드 로직을 구현하는 함수.|

이처럼 Amplify는 스토리지, 함수 등의 기능을 제공하며, 실제로는 그 뒤에서 S3, Lambda와 같은 AWS 서비스를 자동으로 프로비저닝하고 관리합니다.

## 4. 🧹 리소스 정리

실습 후 불필요한 비용 발생을 막기 위해 배포된 리소스를 깨끗하게 정리해야 합니다.

- **Amplify 앱 삭제**: Amplify 콘솔의 **`앱 설정(App settings)`** 메뉴에서 **`일반(General)`** 탭으로 이동하여 앱을 삭제합니다. 이 과정은 Amplify가 배포한 모든 백엔드 리소스를 자동으로 삭제합니다.
    
- **GitHub 리포지토리 삭제**: GitHub에서 실습을 위해 복제했던 리포지토리도 함께 삭제하여 정리합니다.
    

Amplify는 이처럼 웹/모바일 애플리케이션의 개발, 배포, 호스팅, 백엔드 관리까지 모두 통합하여 개발 생산성을 극대화하는 강력한 '올인원' 플랫폼입니다.
