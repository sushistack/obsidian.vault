
Amazon S3 사전 서명된 URL(Pre-signed URLs)은 S3 버킷의 객체에 대해 한정된 시간 동안 접근 권한을 부여하는 URL입니다. 이 기능은 버킷이 비공개(private) 상태인 경우에도 특정 사용자에게 임시로 객체에 대한 접근을 허용할 때 유용하게 사용됩니다.

---

### 1. 사전 서명된 URL의 작동 원리

1. **URL 생성:** S3 객체에 대한 접근 권한을 가진 사용자(버킷 소유자 또는 IAM 사용자)가 콘솔, CLI 또는 SDK를 사용하여 특정 객체에 대한 사전 서명된 URL을 생성합니다.

2. **권한 상속:** 생성된 URL은 URL을 생성한 사용자의 권한을 상속받습니다. 이 URL을 받은 사용자는 URL 생성자의 권한에 따라 `GET` (다운로드) 또는 `PUT` (업로드) 작업을 수행할 수 있습니다.

3. **URL 공유:** URL 생성자는 이 URL을 외부 사용자에게 공유합니다.

4. **임시 접근:** URL을 받은 사용자는 해당 URL을 사용하여 S3 객체에 접근하며, URL에 설정된 만료 시간까지만 유효합니다.


> **예시:** 버킷 소유자가 특정 비디오 파일에 대한 사전 서명된 URL을 생성하여 친구에게 보냅니다. 친구는 이 URL을 통해 버킷에 접근 권한이 없더라도 해당 비디오 파일을 다운로드할 수 있습니다.

---

### 2. 주요 사용 사례

- **비공개 버킷 파일 공유:** 비공개 S3 버킷에 저장된 파일을 특정 사용자에게만 임시로 다운로드할 권한을 부여할 때.

- **보안 업로드:** 웹 애플리케이션에서 사용자가 파일을 업로드할 때, 버킷을 공개하지 않고도 정해진 위치에만 파일을 올릴 수 있도록 임시 업로드 권한을 부여할 때.

- **동적 권한 관리:** 로그인한 사용자에게만 유료 콘텐츠 다운로드 링크를 제공하는 등, 애플리케이션 로직에 따라 동적으로 URL을 생성하여 접근을 제어할 때.

---

### 3. 유효 기간 (Expiration)

- **S3 콘솔:** 최대 **12시간**
- **AWS CLI:** 최대 **168시간** (7일)

사전 서명된 URL은 일시적인 접근을 위한 기능이므로, 반드시 만료 시간을 설정해야 합니다. 만료 시간이 지나면 URL은 더 이상 유효하지 않습니다.
